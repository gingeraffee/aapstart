# AAP Start repo guidance

## Mission
Preserve the working product foundation and improve the design quality.

## Hard rules
- Do not create a new repo.
- Do not rebuild from scratch.
- Do not touch the older Streamlit app.
- Preserve working Google Sheets auth, progress tracking, quiz flow, acknowledgement flow, and routing unless a change is required for UI integration.
- Keep this desktop-first.
- Do not expose track logic to the user in a technical way.
- Do not surface placeholder modules as if they are complete.
- Use the AAP Start prototype CSS/login/portal files as the visual source of truth.

## Working style
- Inspect first, then plan, then implement.
- Prefer the simplest implementation that still feels premium and intentional.
- Stop after the review checkpoint: shell, login, overview, one polished module page, quiz/ack alignment.

## Required output at checkpoint
- Summary of files changed
- What improved
- What still needs work
- Any risks or bugs found
